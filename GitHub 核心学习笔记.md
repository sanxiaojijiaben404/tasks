# GitHub 核心学习笔记

#### 一、核心概念

- 仓库（Repository）：项目文件夹，分公开（Public）（开源）和私有（Private）（保密），存放代码、文档等所有资源。
- 分支（Branch）：主分支默认是 main ，开发时新建分支（如 dev ），避免直接修改主分支代码，完成后合并回去。
- 提交（Commit）：代码修改的“快照”，每次修改后提交，需写清晰说明（如“修复登录按钮bug”），方便回溯。
- 拉取请求（Pull Request, PR）：分支开发完成后，向主分支发起“合并申请”，可多人审核代码，通过后合并。

#### 二、常用操作（命令行/网页版通用）

1. 本地项目关联GitHub
1. - 本地初始化： git init （新建仓库）
2. - 关联远程仓库： git remote add origin 仓库地址 
3. - 提交代码： git add . （添加所有文件）→  git commit -m "首次提交"  →  git push -u origin main （推送到远程主分支）
2. 日常开发流程
- 拉取最新代码： git pull （避免冲突）
- 新建分支开发： git checkout -b 分支名 
- 开发完推送分支： git push origin 分支名 
- 网页端发起PR → 审核通过后合并 → 删除无用分支
3. 解决冲突
- 合并时出现冲突，打开冲突文件，删除 <<<<<<< / ======= / >>>>>>>  标记，保留正确代码，重新提交即可。

#### 三、实用技巧

- Star：收藏喜欢的开源项目，方便后续查看。
- Fork：复制他人项目到自己仓库，可基于此修改（需遵守原项目开源协议）。
- Issues：用于提bug、提需求，或讨论项目问题，是开源项目沟通的核心工具。
